# cher-llm-methodology

Eine strukturierte Methodik für die Zusammenarbeit mit Large Language Models (LLMs) wie ChatGPT in komplexen, mehrschrittigen Projekten.

Ziel dieses Repositories ist es, eine klare, reproduzierbare und nachvollziehbare Arbeitsweise zu entwickeln, mit der LLM-basierte Projekte über längere Zeiträume hinweg stabil, konsistent und dokumentiert durchgeführt werden können.

## Ziel des Projekts

Komplexe Vorhaben – etwa die Entwicklung von Frameworks, die Erstellung umfangreicher Dokumentationen oder die Konzeption von Fragebögen – können mit Unterstützung eines LLMs oft sehr effektiv erarbeitet werden. Ohne festgelegtes Vorgehen treten dabei erfahrungsgemäß aber oft typische Probleme auf:

- Kontextverlust über lange Chat-Verläufe  
- Wissensdrift und inkonsistente Antworten  
- fehlende nachvollziehbare Dokumentation  
- schwer reproduzierbare Ergebnisse  

Dieses Repository stellt eine Methodik bereit, die:

- einen stabilen Projektkontext im LLM etabliert,  
- Ergebnisse und Erkenntnisse in Markdown versioniert,  
- ein integriertes Zusammenspiel von ChatGPT und GitHub ermöglicht,  
- klare Strukturen, Rollen und Abläufe definiert  
- und eine iterative, stetig verbesserbare Arbeitsweise unterstützt.

Die Ergebnisse werden in diesem [Wiki](https://github.com/hrmnns/cher-llm-methodology/wiki) festgehalten.

## Vorgehensweise & Workflow

Diese Methodik basiert auf vier grundlegenden Prinzipien:

### **1. Projektanweisung = Steuerlogik**  
Kurz, präzise, stabil.  
Legt Rollen, Formatvorgaben, Vorgehensweise und Konsistenzregeln fest.

### **2. GitHub = Wissensbasis**  
Markdown-Dateien speichern:
- Methoden  
- Entscheidungen  
- Beispiele  
- Dokumentation  
- Versionierung  

### **3. Zusammenarbeit zwischen ChatGPT und Dokumentation**  
- Arbeiten im Chat → Ergebnisse erarbeiten  
- Dokumentation im Repo → Ergebnisse sichern  
- Wiederholen → iterativer Lernprozess

### **4. Iteration statt Perfektion**  
Alles beginnt klein und wird schrittweise verfeinert.
